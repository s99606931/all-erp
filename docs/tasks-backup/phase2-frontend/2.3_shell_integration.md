# Task 2.3: Shell Layout Integration

## 1. 개요 (Overview)

- **Task ID**: 2.3
- **목표**: `apps/frontend/shell` 애플리케이션에 `libs/shared/ui-design`을 연동하고, **Dashboard Layout** (Sidebar + Header + Main Content)을 적용하여 전체 시스템의 기본 골격을 완성한다.
- **선행 작업**: Task 2.2 (Core Components 완료)

## 2. 상세 요구사항 (Requirements)

1.  **Tailwind 설정 연동**: `shell` 앱이 `libs/shared/ui-design`의 Tailwind 설정을 상속받도록 구성.
2.  **Layout 컴포넌트 구현**:
    - **Sidebar**: 접이식(Collapsible) 메뉴, 네비게이션 링크 포함.
    - **Header**: 사용자 프로필, 테마 토글, 알림 아이콘 포함.
    - **Main Content**: `Outlet`을 통한 페이지 렌더링 영역.
3.  **Global Styles 적용**: `global.css`를 import하여 폰트 및 기본 스타일 적용.

## 3. 기술적 의사결정 (Technical Decisions)

- **App Layout**: `shell` 내부가 아닌 `libs/shared/ui-design`에 `ShellLayout` 컴포넌트를 구현하여 재사용성 확보 (향후 다른 포털 등이 생길 경우 대비).
- **Navigation**: `react-router-dom`의 `NavLink`를 사용하여 활성 상태 스타일링.

## 4. 작업 단위 (Work Units)

1.  **Shell Tailwind 설정**: `apps/frontend/shell/tailwind.config.js` 수정 (Preset 적용).
2.  **Layout 컴포넌트 개발 (`libs/shared/ui-design`)**:
    - `src/lib/layouts/shell-layout.tsx`
    - `src/lib/layouts/sidebar.tsx`
    - `src/lib/layouts/header.tsx`
3.  **Shell App 수정**:
    - `apps/frontend/shell/src/app/app.tsx`에서 `ShellLayout` 적용.
    - `global.css` import.

## 5. 승인 기준 (Acceptance Criteria)

- [ ] Shell 앱 실행 시 사이드바와 헤더가 포함된 레이아웃이 보여야 한다.
- [ ] 디자인 시스템의 Primary 컬러와 폰트가 적용되어야 한다.
- [ ] 반응형 동작 (모바일에서 사이드바 숨김 등)이 동작해야 한다.
