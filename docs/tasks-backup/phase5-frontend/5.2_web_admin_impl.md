- **Task ID**: 5.2_web_admin_impl
- **목표**: 각 마이크로서비스의 기능을 통합하여 사용자가 접근할 수 있는 웹 관리자 콘솔을 완성합니다.
- **관련 로드맵**: Phase 5 - 프론트엔드 통합 및 고도화

## 2. 상세 요구사항 (Requirements)


## 3. 기술적 의사결정 (Technical Decisions)

### 3.1 Micro-frontend 패턴 사용 여부
- **선택지 A**: **Monolithic Frontend** (모든 기능을 하나의 Next.js 앱에)
- **선택지 B**: **Micro-frontend** (서비스별로 별도 프론트엔드 앱)
- **✅ 권장사항**: **Monolithic Frontend** (초기)
    - **이유**: 
        1. **개발 효율성**: 초기 단계에서 코드 공유, 상태 관리, 배포 파이프라인이 단순합니다.
        2. **사용자 경험**: 페이지 이동 시 깜빡임 없는 SPA 경험을 제공하기 유리합니다.
        3. **팀 규모**: 현재 인력 구성상 여러 개의 프론트엔드 앱을 관리하는 오버헤드를 줄여야 합니다.
    - **참고**: 추후 서비스 규모가 커지면 **Nx**의 기능을 활용해 점진적으로 분리할 수 있습니다.

### 3.2 폼 관리
- **✅ 권장사항**: **React Hook Form** + **Zod** (유효성 검사)
    - **이유**: 타입 안전하고 성능이 우수합니다.

## 4. 작업 단위 (Work Breakdown)

1.  **인증 로직 구현**:
    - 로그인 페이지 구현
    - JWT 저장 및 자동 갱신
    - 라우팅 가드 (Middleware 또는 HOC)

2.  **서비스별 페이지 구현**:
    - 인사/급여/예산 등 도메인별 페이지
    - React Hook Form + Zod로 폼 관리

3.  **대시보드 구현**:
    - 위젯 및 차트 구현 (Chart.js 또는 Recharts)
    - 주요 지표 표시

4.  **AI 챗봇 UI 통합**:
    - ai-service와 통신
    - 실시간 답변 표시

5.  **Docker Compose에서 테스트**:
    ```bash
    # 전체 환경 실행 (인프라 + 백엔드 + 프론트엔드)
    cd dev-environment
    docker compose -f docker-compose.infra.yml -f docker-compose.dev.yml up -d
    
    # 브라우저 접속
    open http://localhost:4200
    
    # 테스트
    # 1. 로그인 (http://localhost:4200/login)
    # 2. 대시보드 접속 (http://localhost:4200/dashboard)
    # 3. AI 챗봇 테스트 (http://localhost:4200/ai-chat)
    ```

> **참고**: [Docker-First Workflow Guide](../../guides/docker-first-workflow.md)

## 5. 승인 기준 (Acceptance Criteria)
- [ ] **토큰 재발급**: Docker Compose 환경에서 로그인 후 토큰 만료 시 Refresh Token으로 자동 재발급
- [ ] **권한 관리**: 권한이 없는 페이지 접근 시 403 에러 페이지 표시
- [ ] **AI 챗봇**: AI 챗부7에 질문 시 실시간 답변 표시
- [ ] **Hot Reload**: `http://localhost:4200`에서 소스 코드 수정 시 자동 반영
