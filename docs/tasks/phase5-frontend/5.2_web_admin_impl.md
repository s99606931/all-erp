- **Task ID**: 5.2_web_admin_impl
- **목표**: 각 마이크로서비스의 기능을 통합하여 사용자가 접근할 수 있는 웹 관리자 콘솔을 완성합니다.
- **관련 로드맵**: Phase 5 - 프론트엔드 통합 및 고도화

## 3. 기술적 의사결정 (Technical Decisions)

### 3.1 Micro-frontend 패턴 사용 여부
- **선택지 A**: **Monolithic Frontend** (모든 기능을 하나의 Next.js 앱에)
- **선택지 B**: **Micro-frontend** (서비스별로 별도 프론트엔드 앱)
- **✅ 권장사항**: **Monolithic Frontend** (초기)
    - **이유**: 
        1. **개발 효율성**: 초기 단계에서 코드 공유, 상태 관리, 배포 파이프라인이 단순합니다.
        2. **사용자 경험**: 페이지 이동 시 깜빡임 없는 SPA 경험을 제공하기 유리합니다.
        3. **팀 규모**: 현재 인력 구성상 여러 개의 프론트엔드 앱을 관리하는 오버헤드를 줄여야 합니다.
    - **참고**: 추후 서비스 규모가 커지면 **Nx**의 기능을 활용해 점진적으로 분리할 수 있습니다.

### 3.2 폼 관리
- **✅ 권장사항**: **React Hook Form** + **Zod** (유효성 검사)
    - **이유**: 타입 안전하고 성능이 우수합니다.

## 4. 작업 단위 (Work Breakdown)
1.  로그인 페이지 및 인증 로직 구현 (JWT 저장, 자동 갱신)
2.  라우팅 가드 (Middleware 또는 HOC)
3.  서비스별 페이지 구현 (인사/급여/예산 등)
4.  대시보드 위젯 및 차트 구현
5.  AI 챗봇 UI 통합

## 5. 승인 기준 (Acceptance Criteria)
- [ ] 로그인 후 토큰이 만료되면 자동으로 Refresh Token으로 재발급되어야 한다.
- [ ] 권한이 없는 페이지 접근 시 403 에러 페이지가 표시되어야 한다.
- [ ] AI 챗봇에 질문하면 실시간으로 답변이 표시되어야 한다.
