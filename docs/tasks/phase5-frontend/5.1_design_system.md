# PRD: UI/UX 디자인 시스템 구축

## 1. 개요 (Overview)
- **Task ID**: 5.1_design_system
- **목표**: 일관된 사용자 경험을 제공하기 위한 디자인 시스템과 공통 컴포넌트를 구축합니다.
- **관련 로드맵**: Phase 5 - 프론트엔드 통합 및 고도화

## 2. 상세 요구사항 (Requirements)
- [ ] **공통 레이아웃**: Sidebar, Header(네비게이션), Footer 컴포넌트
- [ ] **UI 컴포넌트 라이브러리**: Button, Input, Table, Modal, Form 등 (Shadcn/UI 활용)
- [ ] **테마 시스템**: Light/Dark 모드 전환 지원
- [ ] **반응형 디자인**: 모바일/태블릿/데스크톱 대응

## 3. 기술적 의사결정 (Technical Decisions)

### 3.1 상태 관리
- **선택지 A**: **Zustand** (경량, 간단)
- **선택지 B**: **Redux Toolkit** (복잡한 상태 관리)
- **선택지 C**: **TanStack Query** (서버 상태 전용)
- **✅ 권장사항**: **Zustand** (전역 상태) + **TanStack Query** (서버 상태)
    - **이유**: Zustand는 간결하고, TanStack Query는 캐싱/리페칭 등 서버 상태 관리에 최적화되어 있습니다.

### 3.2 API Client
- **✅ 권장사항**: **TanStack Query** + **Axios**
    - Swagger로 생성된 API 스펙을 기반으로 타입 안전한 API 클라이언트 자동 생성 (`openapi-typescript-codegen` 활용)

## 4. 작업 단위 (Work Breakdown)
1.  Shadcn/UI 기본 컴포넌트 설치 및 커스터마이징
2.  레이아웃 컴포넌트 (`AppLayout`, `Sidebar`, `Header`) 구현
3.  테마 토글 기능 구현 (CSS Variables 활용)
4.  Zustand Store 설계 (User Info, Theme 등)
5.  TanStack Query 설정 및 공통 Fetcher 함수 작성

## 5. 승인 기준 (Acceptance Criteria)
- [ ] 모든 페이지가 동일한 레이아웃(Sidebar, Header)을 사용해야 한다.
- [ ] 다크 모드 전환 시 모든 컴포넌트의 색상이 자동으로 변경되어야 한다.
- [ ] 공통 컴포넌트 사용 가이드 문서(Storybook 등)가 작성되어야 한다.
