# Task 2.2: Core ERP Components Implementation

## 1. 개요 (Overview)

- **Task ID**: 2.2
- **목표**: ERP 업무의 핵심인 **데이터 그리드(Data Grid)**와 **고성능 폼(Form)** 컴포넌트를 구현한다.
- **선행 작업**: Task 2.1 (Foundation 완료)

## 2. 상세 요구사항 (Requirements)

1.  **ERPDataGrid**:
    - `TanStack Table v8` 기반.
    - 기능: 페이지네이션, 소팅, 클라이언트/서버 사이드 필터링, 컬럼 가시성 토글.
    - 스타일: Compact 모드 지원 (Dense row).
2.  **ERPForm**:
    - `React Hook Form` + `Zod` 기반.
    - `AutoForm` 개념 적용: Zod 스키마만 넣으면 폼이 렌더링되는 컴포넌트 래퍼.
    - Validation Error 메시지 자동 노출.

## 3. 기술적 의사결정 (Technical Decisions)

- **Table Virtualization**: 데이터가 100건 이상일 경우 `TanStack Virtual` 적용 고려 (초기는 제외).
- **Form Layout**: Grid 시스템을 활용하여 반응형 폼 레이아웃 지원 (1열 -> 2열 -> 4열).

## 4. 작업 단위 (Work Units)

1.  **ERPDataGrid 구현**:
    - `libs/shared/ui-design/src/lib/components/data-grid/data-grid.tsx`
    - `libs/shared/ui-design/src/lib/components/data-grid/columns.tsx`
2.  **ERPForm 구현**:
    - `libs/shared/ui-design/src/lib/components/form/form.tsx`
    - `libs/shared/ui-design/src/lib/components/form/form-field.tsx`
3.  **Export 설정**:
    - `libs/shared/ui-design/src/index.ts`에서 컴포넌트 export.

## 5. 승인 기준 (Acceptance Criteria)

- [ ] `ERPDataGrid`에 JSON 배열 데이터를 넘겼을 때 테이블이 렌더링되어야 한다.
- [ ] 테이블 헤더 클릭 시 정렬이 되어야 한다.
- [ ] `ERPForm`에 Zod 스키마와 submit 핸들러를 넘겼을 때 폼이 동작해야 한다.
