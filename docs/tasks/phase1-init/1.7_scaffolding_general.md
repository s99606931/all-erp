# PRD: General Domain 서비스 스캐폴딩

## 1. 개요 (Overview)
- **Task ID**: 1.7_scaffolding_general
- **목표**: 총무/자산 관리의 핵심인 `asset-service`, `supply-service`, `general-affairs-service`의 기본 앱을 생성합니다.
- **관련 로드맵**: Phase 1 - 서비스 스캐폴딩

## 2. 상세 요구사항 (Requirements)
- [ ] **NestJS 앱 생성**: 3개의 마이크로서비스 생성
- [ ] **기본 포트 설정**:
    - `asset-service`: 3031
    - `supply-service`: 3032
    - `general-affairs-service`: 3033
- [ ] **공통 설정 적용**: Health Check, Swagger 설정

## 3. 기술적 의사결정 (Technical Decisions)
> 특이 사항 없음. 표준 스캐폴딩 절차를 따릅니다.

## 4. 작업 단위 (Work Breakdown)
1.  Nx CLI로 각 서비스 생성:
    - `pnpm nx g @nx/nest:app --directory=apps/general/asset-service --name=asset-service --no-interactive`
    - `pnpm nx g @nx/nest:app --directory=apps/general/supply-service --name=supply-service --no-interactive`
    - `pnpm nx g @nx/nest:app --directory=apps/general/general-affairs-service --name=general-affairs-service --no-interactive`
2.  각 서비스 설정:
    - `main.ts`: Swagger, ValidationPipe, Health Check 설정
    - `app.module.ts`: ConfigModule 설정
    - 포트: 3031 (asset), 3032 (supply), 3033 (general-affairs)
3.  **Docker Compose 통합**:
    - `dev-environment/docker-compose.dev.yml`에 3개 서비스 추가
4.  **Docker Compose로 실행 및 검증**

> **참고**: [Docker-First Workflow Guide](../../guides/docker-first-workflow.md)
