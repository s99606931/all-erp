# PRD: Frontend (Web Admin) 스캐폴딩

## 1. 개요 (Overview)
- **Task ID**: 1.9_scaffolding_frontend
- **목표**: 관리자용 통합 웹 콘솔인 `web-admin`을 Next.js 기반으로 생성합니다.
- **관련 로드맵**: Phase 1 - 서비스 스캐폴딩

## 2. 상세 요구사항 (Requirements)
- [ ] **Next.js 앱 생성**: Nx를 사용하여 Next.js 애플리케이션 생성
- [ ] **기본 포트 설정**: 4200
- [ ] **스타일링 설정**: Tailwind CSS 및 Shadcn/UI 초기 설정

## 3. 기술적 의사결정 (Technical Decisions)

### 3.1 라우팅 방식 (Routing)
- **선택지 A**: **App Router** (최신)
    - 장점: React Server Components, Layout RFC 등 최신 기능 활용
    - 단점: 기존 Pages Router와 패러다임이 다름 (러닝 커브)
- **선택지 B**: **Pages Router** (기존)
    - 장점: 안정적, 레퍼런스 많음
- **✅ 권장사항**: **`App Router` (Next.js 15부터 기본값)**
    - **이유**: Next.js 15부터 App Router가 기본 라우팅 방식이며, React Server Components와 Partial Prerendering 등 최신 기능을 완전히 활용할 수 있습니다.

### 3.2 UI 라이브러리
- **✅ 권장사항**: **`Tailwind CSS` + `Shadcn/UI`**
    - **이유**: 커스터마이징이 용이하고 디자인 시스템 구축에 최적화되어 있습니다.

## 4. 작업 단위 (Work Breakdown)
1.  Nx CLI로 Next.js 앱 생성:
    - `pnpm nx g @nx/next:app --directory=apps/frontend/web-admin --name=web-admin --appDir=true --src=false --no-interactive`
2.  Tailwind CSS 설치 (Next.js 생성 시 자동 설치 옵션 선택)
3.  Shadcn/UI 초기화:
    - `pnpm dlx shadcn@latest init`
    - TypeScript: `yes`, Style: `New York`, CSS variables: `yes`
4.  **Docker Compose 통합**:
    - `dev-environment/docker-compose.dev.yml`에 web-admin 서비스 추가
    - 포트: 4200
    - 볼륨 마운트: `../apps/frontend:/workspace/apps/frontend:cached`
5.  **Docker Compose로 실행 및 검증**

> **참고**: [Docker-First Workflow Guide](../../guides/docker-first-workflow.md)

## 5. 승인 기준 (Acceptance Criteria)
- [ ] **Docker Compose로 서비스 시작**: `docker compose -f docker-compose.infra.yml -f docker-compose.dev.yml up -d` 실행 성공
- [ ] **Next.js 기본 페이지 확인**: `http://localhost:4200` 접속 시 정상 표시
- [ ] **Tailwind CSS 동작 확인**: 유틸리티 클래스가 정상적으로 적용
- [ ] **Hot Reload 동작**: 소스 코드 수정 시 자동 반영
