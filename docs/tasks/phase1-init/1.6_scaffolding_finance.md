# PRD: Finance Domain 서비스 스캐폴딩

## 1. 개요 (Overview)
- **Task ID**: 1.6_scaffolding_finance
- **목표**: 재무/회계 관리의 핵심인 `budget-service`, `accounting-service`, `settlement-service`의 기본 앱을 생성합니다.
- **관련 로드맵**: Phase 1 - 서비스 스캐폴딩

## 2. 상세 요구사항 (Requirements)
- [ ] **NestJS 앱 생성**: 3개의 마이크로서비스 생성
- [ ] **기본 포트 설정**:
    - `budget-service`: 3021
    - `accounting-service`: 3022
    - `settlement-service`: 3023
- [ ] **공통 설정 적용**: Health Check, Swagger 설정

## 3. 기술적 의사결정 (Technical Decisions)

### 3.1 정밀도 처리 (Decimal Handling)
- **권장사항**: 재무 데이터는 부동소수점 오차를 방지하기 위해 `number` 타입 대신 `Decimal` 라이브러리(Prisma Decimal 등)를 사용해야 합니다. 스캐폴딩 단계에서는 관련 유틸리티 라이브러리(`libs/shared/util-decimal` 등) 사용을 고려하여 구조를 잡습니다.

## 4. 작업 단위 (Work Breakdown)
1.  `pnpm nx g @nx/nest:app budget-service --directory=apps/finance/budget-service`
2.  `pnpm nx g @nx/nest:app accounting-service --directory=apps/finance/accounting-service`
3.  `pnpm nx g @nx/nest:app settlement-service --directory=apps/finance/settlement-service`
4.  각 서비스 `main.ts` 포트 및 기본 설정 적용
