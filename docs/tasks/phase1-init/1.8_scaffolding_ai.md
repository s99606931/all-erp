## 3. 기술적 의사결정 (Technical Decisions)

### 3.1 AI 서비스 언어 (Language)
- **선택지 A**: **Node.js (NestJS)** + LangChain.js
    - 장점: 기술 스택 통일, 유지보수 용이
    - 단점: Python 생태계(PyTorch, Pandas 등) 활용 제한적
- **선택지 B**: **Python (FastAPI)**
    - 장점: AI/ML 라이브러리 활용 최적
    - 단점: 기술 스택 파편화, 배포 파이프라인 별도 구성 필요
- **✅ 권장사항**: **`NestJS` (Main) + `Python` (Worker/Sidecar)**
    - **이유**: API 게이트웨이 역할과 비즈니스 로직은 NestJS로 통일성을 유지하고, 실제 무거운 AI 연산이나 Python 라이브러리가 필수적인 작업은 별도의 Python 컨테이너나 Serverless 함수로 분리하여 호출하는 구조를 권장합니다. 초기 스캐폴딩은 **NestJS**로 생성합니다.
