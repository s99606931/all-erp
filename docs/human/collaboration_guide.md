# AI 협업 가이드 및 작업 지시 템플릿

## 1. 개요
이 문서는 **사람(Human)**과 **AI(Gemini)**가 MSA ERP 프로젝트를 진행하면서 효율적으로 협업하기 위한 절차와 지침을 정의합니다. 특히, 사람이 AI에게 명확하고 정확하게 작업을 지시하여 최상의 결과를 얻는 것을 목표로 합니다.

## 2. 협업 프로세스 (Collaboration Workflow)
각 개발 단계에서 사람과 AI의 역할을 명확히 구분합니다.

| 단계 | 사람 (Human)의 역할 | AI (Gemini)의 역할 |
|---|---|---|
| **1. 기획 & 설계** | - 요구사항 정의 및 도메인 분석<br>- 핵심 아키텍처 결정<br>- 기술 스택 승인 | - 기술 조사 및 대안 제안<br>- 아키텍처 초안 및 다이어그램 작성<br>- DB 스키마 초안 설계 |
| **2. 구현 (Coding)** | - **작업 지시 (Prompting)**<br>- 복잡한 비즈니스 로직 설계<br>- 코드 리뷰 및 품질 관리 | - 보일러플레이트 코드 생성<br>- 단위/기능 구현<br>- 리팩토링 및 최적화 제안 |
| **3. 테스트 & 디버깅** | - 테스트 시나리오 정의<br>- 예외 케이스 발굴<br>- 최종 승인 (Merge) | - 테스트 코드(Unit/E2E) 작성<br>- 에러 로그 분석 및 버그 수정<br>- 엣지 케이스 처리 로직 추가 |
| **4. 문서화** | - 문서 구조 잡기<br>- 최종 검수 | - API 문서(Swagger) 자동화<br>- README 및 개발 가이드 작성 |

## 3. AI 작업 지시 템플릿 (Task Instruction Templates)
AI에게 작업을 지시할 때 아래 템플릿을 복사하여 내용을 채워 사용하세요.

### 3.1 신규 기능 구현 지시 (Feature Implementation)
```markdown
# [작업 지시] 신규 기능 구현: {기능명}

## 1. 목표 (Goal)
- {무엇을 구현하는지, 어떤 문제를 해결하는지 한 줄 요약}

## 2. 상세 요구사항 (Requirements)
- [ ] {기능 요구사항 1}
- [ ] {기능 요구사항 2}
- [ ] {예외 처리 요구사항}

## 3. 입력 및 출력 (I/O)
- **Input**: {API 요청 파라미터, 함수 인자 등}
- **Output**: {API 응답 포맷, 리턴 값 등}

## 4. 참고 파일 및 컨텍스트 (Context)
- 참고 파일: `@{파일명}`
- 관련 테이블: `{테이블명}`

## 5. 제약 사항 (Constraints)
- `coding_convention.md`를 준수할 것.
- {특정 라이브러리 사용 여부 등}
```

### 3.2 버그 수정 지시 (Bug Fix)
```markdown
# [작업 지시] 버그 수정: {버그명}

## 1. 현상 (Symptoms)
- {어떤 상황에서 어떤 에러가 발생하는지 설명}
- 에러 로그:
  ```
  {에러 로그 붙여넣기}
  ```

## 2. 예상 원인 (Hypothesis)
- {의심가는 부분이나 파일이 있다면 기재}

## 3. 수정 방향 (Direction)
- {어떻게 수정했으면 좋겠는지, 혹은 원인 분석부터 필요한지 명시}
```

### 3.3 리팩토링 지시 (Refactoring)
```markdown
# [작업 지시] 코드 리팩토링

## 1. 대상 파일
- `@{파일명}`

## 2. 리팩토링 목적
- [ ] 가독성 향상
- [ ] 성능 최적화
- [ ] 중복 코드 제거
- [ ] 구조 개선 (예: Service 레이어 분리)

## 3. 요청 사항
- 기존 로직의 동작은 유지할 것 (Behavior Preservation).
- 변경 전후의 차이점을 설명할 것.
```

## 4. 효과적인 프롬프트 작성 팁 (Tips)
1.  **구체적인 컨텍스트 제공**: 단순히 "코드 짜줘"보다는 "현재 `user.service.ts`에 있는 `login` 함수를 JWT 기반으로 수정해줘"와 같이 구체적으로 지시합니다.
2.  **참고 파일 명시**: 작업에 필요한 파일이 있다면 `@` 멘션 기능을 적극 활용하여 AI가 해당 파일을 읽게 합니다.
3.  **단계별 지시 (Chain of Thought)**: 복잡한 작업은 한 번에 시키지 말고, "먼저 계획을 세워줘", "그 다음 코드를 작성해줘"와 같이 단계를 나눕니다.
4.  **피드백 루프**: AI의 결과물이 마음에 들지 않으면, 구체적으로 어떤 부분이 부족한지 피드백을 주어 수정하게 합니다.
